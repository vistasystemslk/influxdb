THIS VERSION OF THE SOFTWARE USES INFLUXDB WITH TELEGRAF, THAT USING ESP32 TO SEND DATA IA MQTT BROKER WHICH IS IN NETERNET, 
THE MQTT USES QOS1 ARCHITECTURE ALLOWING BUFFER THE DATA IN CASE OF SERVER( INFLUX/TELEGRAF) FAILURE,

TO GET STARTED, YOU SHOULD ENTER YOUR WIFI DATA AND LOGIN DATA, URL OF MQTT BROKER IN ESP32 CODE. /ESP32 CODE USES PLATFORM.IO INSIDE VSCODE
NEXT STEP IS TO LAUNCHING INFLUEXDB. AFTER REGISTRATION THE TOKEN NEED TO SAVE TO BE USED FOR TELEGRAF CONFIGURATION.
AFTER ACQUIRING TOKEN, TELEGRAF.CONFIG FILE NEED TO MODIFY GIVING PARAMETERS FOR 
    * token, organization, bucket, server, topic, inputs.mqtt_consumer.topic_parsing need to edit correspondingly.

THEN RUNNING DOCKER CONTAINER docker compose up -build will turn on the docker container, then by login into http://localhost:8086/ influxdb dashboard will appear
